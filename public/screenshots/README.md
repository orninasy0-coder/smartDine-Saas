# PWA Screenshots

This directory contains screenshots for the Progressive Web App (PWA) install prompt.

## Required Screenshots

### Mobile (Narrow Form Factor)
- **Size**: 540x720 pixels (minimum)
- **Aspect Ratio**: 3:4
- **Format**: PNG or JPEG
- **Files**:
  - `menu-view.png` - Mobile menu browsing view

### Desktop (Wide Form Factor)
- **Size**: 1280x720 pixels (minimum)
- **Aspect Ratio**: 16:9
- **Format**: PNG or JPEG
- **Files**:
  - `dashboard.png` - Restaurant owner dashboard view

## Screenshot Guidelines

1. **Content**: Show actual app functionality
2. **Quality**: High-resolution, clear images
3. **Branding**: Include app branding elements
4. **Context**: Show realistic use cases
5. **Text**: Avoid excessive text, focus on visuals

## Best Practices

- Use real data (not lorem ipsum)
- Show the app in action
- Highlight key features
- Use consistent styling
- Ensure good contrast and readability

## Capturing Screenshots

### For Mobile View
1. Open the app in Chrome DevTools mobile view
2. Set viewport to 540x720
3. Navigate to the menu page
4. Take screenshot (Ctrl+Shift+P → "Capture screenshot")

### For Desktop View
1. Open the app in browser
2. Set viewport to 1280x720
3. Navigate to the dashboard
4. Take screenshot
